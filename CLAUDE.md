# CLAUDE.md — Relay

Relay is an open-source, self-hosted team-chat platform (Slack alternative) built with Go microservices and a single Flutter codebase for all 6 platforms.

---

## Repo layout

```
relay.gg/
  apps/
    flutter/          Single Flutter codebase → iOS, Android, Web, Windows, macOS, Linux
  services/           9 Go microservices (one binary per directory)
    auth/             JWT issue/refresh, Argon2id, TOTP, OAuth
    api/              REST gateway — only service exposed to the internet
    messaging/        WebSocket hub, NATS publisher, presence, typing
    notification/     Push (FCM/APNS) + email dispatch
    search/           Meilisearch indexing + query
    file/             MinIO upload/download, presigned URLs, thumbnails
    integration/      Webhooks, slash commands, Socket Mode, app registry
    media/            Livekit control plane — rooms, TURN credentials
    federation/       Cross-server S2S HTTP with Ed25519 signing
  packages/
    events/           Shared NATS event types (Go) — import in every service
    serviceutil/      Shared Go middleware: JWT auth, NATS connect, auth client
    proto/            Protobuf stubs for internal gRPC (auth + messaging)
    sdk-go/           Public Go SDK for bot developers
    sdk-dart/         Public Dart SDK for bot developers
  infra/
    docker/           docker-compose.yml — local infra only (no services)
  docs/               Architecture, dev-plan, gap-analysis, integration guide
  .scratch/           Gitignored — research and design scratch files
```

## Go workspace

The repo uses a Go workspace (`go.work`). All services and packages are linked — you do **not** need to publish packages to use them across services. After adding a new package, add it to `go.work`:

```
use ./packages/your-new-package
```

### Shared packages — always import these instead of reimplementing

```go
// Shared NATS event types — use subject constants and typed structs
import "github.com/therelayproject/relay/packages/events"

// NATS connection factory (reads NATS_URL env var)
import "github.com/therelayproject/relay/packages/serviceutil/natsconn"

// JWT auth middleware for HTTP handlers
import "github.com/therelayproject/relay/packages/serviceutil/middleware"

// Auth Service HTTP client (for WebSocket upgrade paths only)
import "github.com/therelayproject/relay/packages/serviceutil/authclient"
```

## Adding a new inter-service event

1. Add the event struct + subject constant to `packages/events/`.
2. Publisher: `nc.Publish(events.SubjectFoo, data)`.
3. Subscriber: `nc.Subscribe(events.SubjectFoo, handler)`.
4. Keep event structs **append-only** — never remove or rename fields.

## Database

- PostgreSQL 16. All IDs are **Snowflake bigints** generated by `next_snowflake()`.
- Migrations live in `services/api/migrations/` and are numbered `000001_`, `000002_`, etc.
- Every migration must have a matching `.down.sql` rollback file.
- Migrations must be **idempotent** (`IF NOT EXISTS`, `IF NOT EXISTS` on indexes).
- Run `make migration name=<snake_case_name>` to create a new pair.

### Key schema facts

- `messages.blocks JSONB` — Block Kit payload; `content` is fallback text.
- `workspace_members.role` — ENUM `('owner','admin','member','guest')`.
- `channel_members.last_read_id` — Snowflake cursor for unread tracking.
- `workspace_domain_policies.classification` — ENUM `('internal','external','blocked')`.
- Soft deletes on messages: `deleted_at TIMESTAMPTZ` (never hard-delete in Phase 0–2).

## Go service conventions

Every service follows this structure:

```
services/<name>/
  cmd/main.go           HTTP server, env var config, graceful 30s SIGTERM shutdown
  internal/
    handler/            HTTP handlers (one file per domain)
    store/              DB queries (no raw SQL in handlers)
    bus/                NATS publish helpers
    subscriber/         NATS subscribe + dispatch
  Dockerfile            Multi-stage: golang:1.22-alpine builder → alpine:3.19
  go.mod
  README.md
```

- Use `zerolog` for structured logging (`github.com/rs/zerolog`).
- Read config from environment variables only — no config files.
- Wrap the HTTP mux with `middleware.RequireAuth(secret)` on every authenticated route.
- Use `middleware.ClaimsFrom(ctx)` inside handlers to get `userID`, `workspaceID`, `role`.
- The API Service (`:8081`) is the **only** service the internet touches. Internal service HTTP calls use Docker hostnames (e.g., `http://auth:8080`).

## Coding principles

### Go — decisions

**Error handling**
- Wrap with `fmt.Errorf("operation description: %w", err)` — stdlib only, no `pkg/errors`
- Declare sentinel errors as package-level vars: `var ErrNotFound = errors.New("not found")`
- Callers check with `errors.Is` / `errors.As` — never string-match error messages
- Never `panic` in request handlers — always return errors up the call stack

**Logging (zerolog)**
- Always use structured fields: `log.Error().Str("user_id", uid).Err(err).Msg("failed to send message")`
- Log only at handler/entry-point level — `internal/store/` and `internal/service/` return errors, never log
- Levels: `Debug` = dev trace, `Info` = lifecycle events, `Warn` = recoverable, `Error` = failures
- Never log JWT values, passwords, tokens, or any secret material

**Interfaces**
- Define interfaces at the **consumer** (caller), not the implementer
- Maximum 3 methods per interface — split larger ones
- Only create an interface when you have ≥2 concrete implementations or need to mock in tests
- Name with `-er` suffix for single-action interfaces: `MessageSender`, `TokenValidator`

**Testing**
- Use `testify` (`github.com/stretchr/testify`) for `assert` and `require`
- Use `testify/mock` for mocking interfaces
- Table-driven tests for all pure/deterministic functions
- No real network, DB, or NATS calls in unit tests — mock all external deps via interfaces
- Test files: `package foo_test` (black-box) for public API, `package foo` (white-box) for internals

**HTTP handlers**
- Stdlib shape: `func (h *Handler) Name(w http.ResponseWriter, r *http.Request)` — no return value
- Each service's `internal/handler/` has shared helpers: `writeJSON(w, status, v)` and `writeError(w, status, msg)`
- All error responses use the same JSON shape: `{"error": "human readable message"}`
- All endpoints under `/api/v1/` — never bare `/api/`

**Database**
- Driver: `pgx/v5` (`github.com/jackc/pgx/v5`) with `pgxpool` for connection pooling — no ORM, no sqlx
- All SQL lives in `internal/store/` — no queries in handlers or the service layer
- Transactions: `pool.BeginTx(ctx, pgx.TxOptions{})`, always `defer tx.Rollback(ctx)` before `tx.Commit(ctx)`
- Scan into explicit structs — never `interface{}` or `map[string]interface{}`

**Constants**
- No magic numbers anywhere — name every timeout, port, limit, and cache size as a typed constant

---

### Dart / Flutter — decisions

**State management (Riverpod)**
- `AsyncNotifier` for anything touching the network or Drift DB
- `Notifier` for pure synchronous local UI state
- No `setState` in feature code — only allowed in small, truly self-contained leaf widgets
- All providers use `@riverpod` code generation — no manual `NotifierProvider(...)` calls
- Colocate providers with their feature: `lib/features/<feature>/provider.dart`

**Widget decomposition**
- Extract to a named `class` when a widget's `build` method exceeds ~50 lines
- No anonymous `Builder` / `Consumer` widgets nested more than 2 levels deep
- Default to `ConsumerWidget`; use `ConsumerStatefulWidget` only when `initState`, `dispose`, or an `AnimationController` is needed
- One public widget class per file

**Models**
- `fromJson` factory constructor on every API response model
- Use `freezed` for immutable domain models — gets `copyWith`, `==`, `hashCode` for free
- Never pass a Drift `DataClass` directly to a widget — map it to a domain/UI model first
- All server IDs are `String` in Dart — never cast a Snowflake ID to `int`

**Async**
- Never `await` inside `build()` — ever
- All network and DB calls live inside `AsyncNotifier` methods or functions they call
- Always handle all three states explicitly: `.when(data: ..., loading: ..., error: ...)` — no silent failures

---

### Cross-cutting

- **Security**: All user-supplied input is untrusted until explicitly validated. JWT secret ≥32 bytes enforced at startup. Never log secrets.
- **API versioning**: All endpoints under `/api/v1/`. A versioned endpoint is never broken — add `/api/v2/` instead.
- **NATS events**: Event structs are append-only. Never remove or rename a field — add new optional fields only.

## Flutter conventions

- State management: **Riverpod 2.x** — use `@riverpod` annotation + code generation.
- Navigation: **go_router** — all routes defined in `lib/app/router.dart`.
- Local cache: **Drift** (SQLite) — schema in `lib/data/database.dart`, 500-message eviction window per channel.
- IDs from the server are stored as `String` in Dart — never cast Snowflake IDs to `int` (JSON precision loss).
- Build flavors via `--dart-define=FLAVOR=dev|staging|prod`. Use `AppConfig.apiUrl` / `AppConfig.wsUrl` everywhere — never hardcode URLs.
- External/federated users get an amber badge using `RelayColors.external` (`#FAA61A`).

### Drift code generation

After changing any table in `lib/data/tables/`, regenerate:

```bash
cd apps/flutter && dart run build_runner build --delete-conflicting-outputs
```

## CI/CD

GitHub Actions (`.github/workflows/ci.yml`):

| Job | Trigger | What it does |
|-----|---------|--------------|
| `lint-go` | PR + push | `golangci-lint` on all services + packages |
| `test-go` | PR + push | `go test -race` with postgres + redis service containers |
| `lint-dart` | PR + push | `dart format` check + `flutter analyze` |
| `test-dart` | PR + push | `flutter test --coverage` |
| `build-images` | PR + push | Build all 9 Docker images; **push to GHCR on `main` only** |

Images are pushed to `ghcr.io/therelayproject/relay/<service>:latest` and `:<git-sha>`.

Renovate bot (`renovate.json`) runs weekly (Monday before 6 AM) and auto-merges minor/patch bumps for Go, Dart, and GitHub Actions.

```

chore(deps): update zerolog to v1.33.0
```
